hexo learning

在这个过程中使用的将会是markdown语法
这几行使用的是引用
同样也添加了文章头之类的信息

命令等

文件渲染声称 hexo g
本地文件上线 hexo s
远程文件部署 hexo d

主配置文件参数

默认基本信息都已经包含
参数大全

标签插件

tag_plugins
标签插件是集齐一些类似于html tag的作用，插入一些格式化文本，或者是非文本资源

资源文件夹

资源文件就比如一些图片之类的文件，默认的资源文件夹是source下面除了文章以外的文件
所以对于本地图片或者是一些视频之类的文件可以放在本地一个统一的文件夹下面来进行访问

  • 集中管理资源文件
  • 文章资源文件夹

当使用了文章资源文件夹之后，对资源的访问路径可能会有些麻烦，因此使用相对路径引用标签

使用markdown语法的相对路径引用并不强于上面的引用标签

git相关

在主配置文件下面设置git链接的仓库文件名称
如果git没有相关配置，则会需要输入用户名和密码
否则直接使用git的config配置，来连接远程仓库，包括rsa免密认证

主题相关

主题等在文件夹theme下面，文件夹名称就是主题名
将可用的主题名在根文件夹下面的_config.yml中修改即可
